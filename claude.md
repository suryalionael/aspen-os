# Aspen OS

## Mission

Build the simplest and most enjoyable project operating system for nonprofit and community organizations.

Aspen OS helps organizations manage projects, tasks, deadlines, and collaboration without the complexity of enterprise software.

## Vision

Create a modern alternative to spreadsheets, email threads, and fragmented project management tools.

Aspen OS should feel as intuitive as Notion, as fast as Linear, and as practical as Monday.

## Target Users

Primary:

* Nonprofit organizations
* Volunteer-driven organizations
* Community organizations
* Arts and cultural organizations

Secondary:

* Small teams
* Student organizations
* Social impact initiatives

## Current Phase

MVP

## MVP Features

* Authentication
* Workspaces
* Projects
* Tasks
* Kanban View
* Calendar View
* Table View
* List View
* My Work Dashboard

## Non Goals

Do NOT build:

* CRM
* AI Assistant
* Time Tracking
* Payroll
* Billing
* Complex Automations
* Advanced Reporting
* Mobile App

until MVP adoption is validated.

## Product Principles

1. Simplicity over flexibility.
2. Fast over feature-rich.
3. Opinionated over configurable.
4. Beautiful by default.
5. Desktop-first, mobile-friendly.
6. Every feature must solve a real user problem.

## Definition of Done

A feature is complete only if:

* Works on desktop
* Works on mobile
* Has loading states
* Has empty states
* Has error states
* Uses TypeScript correctly
* Has responsive design
* Is production ready

## Team Rules

No feature may be added unless it solves a real problem observed from users.

Avoid overengineering.

Ship small improvements frequently.

Always optimize for usability and speed.
